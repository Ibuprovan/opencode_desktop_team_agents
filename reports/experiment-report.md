## 3. 核心代码说明

### 3.1 统一接口设计

所有排序算法遵循统一的函数签名：

```python
def algorithm_name(arr: List[Any], stats: Optional[Dict] = None) -> List[Any]
```

- **输入**：待排序列表 + 可选统计字典
- **输出**：排序后的新列表（不修改原列表）
- **副作用**：通过 stats 字典返回比较次数和交换次数

### 3.2 快速排序优化

采用**三数取中法**选择基准元素，减少最坏情况发生的概率：

```python
def _median_of_three(low: int, high: int) -> int:
    mid = (low + high) // 2
    if result[low] > result[mid]:
        result[low], result[mid] = result[mid], result[low]
    if result[low] > result[high]:
        result[low], result[high] = result[high], result[low]
    if result[mid] > result[high]:
        result[mid], result[high] = result[high], result[mid]
    result[mid], result[high] = result[high], result[mid]
    return high
```

### 3.3 冒泡排序优化

添加提前终止标志，当数组已有序时提前退出：

```python
swapped = False
for j in range(0, n - i - 1):
    if result[j] > result[j + 1]:
        result[j], result[j + 1] = result[j + 1], result[j]
        swapped = True
if not swapped:
    break
```

### 4.3 预期性能表现

根据算法理论复杂度，预期在大规模数据上的性能排名：

1. **快速排序** - 平均 O(n log n)，实际表现最优
2. **归并排序** - 稳定 O(n log n)，但空间开销较大
3. **插入排序** - 小规模数据或近似有序数据表现良好
4. **选择排序** - O(n²)，交换次数少
5. **冒泡排序** - O(n²)，比较和交换次数最多

### 4.4 边界情况处理

- **空数组**：所有算法返回空列表 `[]`
- **单元素数组**：直接返回原数组
- **重复元素**：正确处理，保持排序稳定性（冒泡、插入、归并）
